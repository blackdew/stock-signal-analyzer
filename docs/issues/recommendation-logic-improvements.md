# [Improvement] 주식 추천 로직 고도화: V3 루브릭 정합성 및 섹터 편중성 제어

**라벨(제안):** `enhancement`, `ranking`, `risk-management`
**관련 파일:** `src/agents/analysis/ranking_agent.py`
**발견 경위:** 추천 로직 개선점 검토 (`recommendation_logic_improvements.md`)

---

## 현상 및 개선 필요성 (Summary)

현재 `RankingAgent`가 최종 Top 5 종목을 선정하는 방식은 다음과 같은 두 가지 주요 한계점을 지니고 있습니다.

1. **V3 루브릭 가중치 불정합 (Legacy 랭킹 산출식)**:
   - 최신 V3 루브릭은 가치 투자의 요체인 **밸류에이션(Valuation, 20점 만점)** 카테고리를 독립시켰으나, Top 5 랭킹 산출 로직(`select_final_top5`)은 V1/V2 기준으로 작성되어 오직 `supply`와 `fundamental`만 가중 보정(각 15%씩)하고 있습니다.
   
2. **섹터 편중성 리스크 (Sector Concentration Risk)**:
   - 단순 정렬 후 상위 5개를 채우기 때문에 특정 섹터(예: 반도체, 바이오)가 강세일 때 추천 종목 전체가 동일 섹터로 쏠리는 변동성 위험이 있습니다.

## 해결 방안 (Proposed Fix)

### 1. 랭킹 산출식 동적 V3 대응
- 루브릭 V3가 활성화된 경우 (`use_v3=True`), 최종 Top 5 산출 공식을 다음과 같이 개편합니다:
  - **총점 가중치**: 70% → 60%
  - **수급(Supply) 가중치**: 15% (15점 만점 기준 정규화)
  - **밸류에이션(Valuation) 가중치**: 15% 추가 (20점 만점 기준 정규화)
  - **성장성(Fundamental) 가중치**: 15% → 10% (15점 만점 기준 정규화)
- V1/V2 가동 시에는 기존의 `총점 70% + 수급 15% + 펀더멘털 15%` 계산식을 그대로 유지하여 하위 호환성을 확보합니다.

### 2. 섹터 편중 방지 가드 (Sector Concentration Cap) 도입
- 최종 Top 5 구성 시 **동일 섹터 내 최대 2개 종목까지만 허용**하는 제어 로직을 구현합니다.
- 만약 특정 섹터 쏠림으로 인해 5개를 채우지 못하는 비상 상황을 대비해, 제한에 걸리지 않는 종목을 먼저 선정한 후 잔여석은 제한 없이 점수 차순위로 보충하는 Fallback 로직을 장착합니다.

## 검증 계획 (Verification)

1. `tests/agents/analysis/test_ranking_agent.py` 내에 V3 가중치 동작과 섹터 편중 방지 필터링 동작을 검증하는 단위 테스트를 추가합니다.
2. 백테스트 및 전체 분석 시나리오를 구동하여 최종 Top 5 종목이 다각화되면서도 우량한 밸류에이션 종목 위주로 선정되는지 확인합니다.
